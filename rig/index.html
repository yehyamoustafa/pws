Overview

The Rig is an internal document operations dashboard concept designed to centralize document intake, approvals, and audit-ready tracking. The goal was to reduce tool-switching and give operations teams a single place to manage status, ownership, and compliance visibility.

My role: Information architecture, UX flows, design system patterns, high-fidelity UI, and interaction states for a data-dense product.

Timeline: 2–3 weeks

Platform: Web dashboard

Deliverables: IA, key flows, UI system, core screens

Tools: Figma

The problem

Document workflows were scattered across folders, manual trackers, and back-and-forth messages. It was difficult to find the latest file, know who owns the next step, and generate audit-ready reporting when needed.

Goals

  • Centralize intake, approvals, and status in one dashboard
  • Keep dense tables readable with clear hierarchy and patterns
  • Improve visibility: ownership, timelines, and audit trail
  • Create reusable components that scale across workflows

Process

1) Information architecture

Mapped document lifecycle stages and grouped actions by real ops tasks.

2) Data-dense UI patterns

Designed tables, filters, status chips, and bulk actions for speed and clarity.

3) System + states

Built reusable components and states to support scaling and consistency.

Key screens

Key moments across the workflow, focused on intake visibility, approvals, and audit-ready tracking.

Outcome

A centralized operations dashboard that streamlines approvals, improves visibility across document status and ownership, and keeps audit-ready tracking clean, fast, and reliable. The UI system is designed to stay readable even in a dense, ops-heavy environment.
